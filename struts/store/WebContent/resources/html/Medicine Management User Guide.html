  5. Medical Representative Management

User Management

            New user is created from this screen. User role is assigned from this screen to control fine grain access to the application. All urls those hit the application are filtered and checked if the user has access to the url.

 

            Three roles are associated in the application e.g. Super Admin, Admin and End User.

            Super Admin Role

                        Users having Super Admin role called Super Admin user can create new users. A Super Admin user can not be deleted from the application. A Super Admin user can update the medical store details in the application. Only Super Admin user can see the monetary transactions from Transaction Management module in the application.

            Admin Role

                        Users having Admin role called Admin user. This user has lesser access than Super Admin user in the application and more access than End Users.

            End User Role

                        Users having End User role is called End User. A shop may have multiple users accessing the application. Typically there will be only one super admin users and many end users who have restricted access in the application.

 

User can be searched from the below User Management screen. Search is based on user name or user role.

 

 

New user can be added or existing user can be modified from the below screen.

 

 

Few important business rules

1.      A super admin role can not be changed to admin or end user role.

2.      Super admin user can not be deleted.

3.      End user does not get access to the Transaction/User management module. He can not update Store details as well.

4.      User Security is not compromised. If a user enters a url which is not allowed to access by him then the below screen is displayed.
